## EAS: project slug must match `app.json`

If `eas build` fails with *Slug for project identified by extra.eas.projectId does not match the slug field*, open [expo.dev](https://expo.dev) → your project (ID `33268daf-498a-4133-ace0-6846aba4011c`) → **Settings** → set the project **slug** to **`makrion`** so it matches [`app.json`](app.json) `expo.slug`. There is no CLI rename; the dashboard must match the repo.
